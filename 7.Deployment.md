# Deployment
- Make account on AWS.
- Make EC2 free version instance.
- Generate .pem secret key file and download it.
- Change permission of secret key.
```bash
$ cd Downloads
$ chmod 400 <filename>-secret.pem # generated secret file.
```
- Login to instance, every time we need to access the deployed instance we need to use this command first.
```bash
$ ssh -i <Secret-key-file> ubuntu@ec2-3-110-30-144.ap-south-1.compute.amazonaws.com
```
- Now we are in the terminal of the machine now we have to deploy the application.
- Now use curl command of linux to install nodejs in the machine, this is not your pc this is instace that you have made so there is no nodejs to run it and also the version on your pc should match the version you are installing in machine.
- Now clone your project (frontend and backend) from github to machine.
## Frontend
- Now to deploy frontend cd to your frontend file.
- Now run the command to install dependencies.
```bash
$ npm install
```
- We will use `NGINX` to host our frontend project.
```bash
$ sudo apt update # to update packages.
$ sudo apt install nginx
$ sudo systemctl start nginx # to start nginx immediately for current session
$ sudo systemctl enable nginx # to start nginx automatically when server boots up.
```
- Copy code from dist(build files) to /var/www/html/.
```bash
$ sudo scp -r dist/* /var/www/html/
# sudo for root level permissions.
# scp is copy command.
# -r is recursively copy all files.
# dist/* here `/*` is to select all files and folder under dist.
```
- Enable port 80 on your instance on AWS in security->inbound rules->Add rule this is because nginx has default port 80.
## Backend
- First you have to add IP of machine to your database so that your code can connect database from machine also or you can either add 0.0.0.0/0 by which your database is accessible from any IP but it's a bad way and it envolves high risk.
- Now you also need to enable your backend port let's say 3000 to your AWS instance.
- Now cd to backend file.
- Now we can just use `npm run start` to run the backend but there is a problem that when terminal close project also close and we need to run it forever.
- We will use pm2 (process manager) package to make it run forever.
```bash
$ npm i pm2 -g # with -g we do not need to install it for every new project.
$ pm2 start npm -- start # -- start is the script we want to run from the project.
```
#### Some pm2 commands
```bash
$ pm2 logs # for logs.
$ pm2 flush # to clean logs.
$ pm2 list # for the list of processes running.
$ pm2 stop < name > # to stop the process, npm is default name.
$ pm2 delete < name > # to delete the process.
$ pm2 start npm --name "< custom name >" -- start # to start pm2 process with custom name.
```
- Now your backend is also hosted but frontend is calling api on localhost so now we need that frontend should call it on the address backend is hosted and also we need to map port to `/api` for better practices.
- First to map /api we need nginx and add proxy pass code into it
```bash
$ sudo nano /etc/nginx/sites-available/default # to open nginx config file
```
- Inside config file change server_name `_` to address aws provided or if you have any domain name.
- Add configuration rule below it.
```
server_name 3.110.30.144 #change server name if you have domain name

location /api/ {
    proxy_pass http://localhost:3000/;
    
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_cache_bypass $http_upgrade;
}
```
- Now restart nginx
```bash
$ sudo systemctl restart nginx
```
- Make sure that in your frontend project the base url is /api/ so that frontend automatically takes the address on which its hosted.
